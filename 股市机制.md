# 玄剑虚拟股市机制深度剖析

## 一、市场架构

### 1.1 核心参与者

| 角色 | 功能 | 实现方式 |
|:---|:---|:---|
| **投资者** | 买卖股票、持有资产 | 所有注册用户 |
| **发行方** | 创建股票、设定初始参数 | 管理员 |
| **做市商** | 提供流动性、平滑价格波动 | 系统自动执行 |
| **监管者** | 设定规则、维护公平 | 代码层面的硬约束 |

### 1.2 资产体系

```
贡献点（货币）
    ↓ 签到、活动获取
    ↓ 交易消耗
股票（资产）
    ├── 基础属性：代码、名称、描述
    ├── 价格属性：基础价、当前价
    ├── 供给属性：总股本、流通股
    └── 波动属性：波动率、趋势值
```

## 二、价格形成机制

### 2.1 传统模型的问题

早期虚拟股市采用**纯随机模型**：
```
新价格 = 当前价格 + 当前价格 × 随机因子
```

**致命缺陷**：
- 价格与交易行为完全脱钩
- 用户买入后价格可能因随机因素下跌
- 无法形成供需博弈
- 缺乏真实市场的"发现价格"功能

### 2.2 交易驱动模型

本系统采用**混合驱动模型**，价格由三股力量共同决定：

```
价格变动 = 交易驱动因子 × 70% + 随机因子 × 20% + 趋势因子 × 10%
```

#### 2.2.1 交易驱动因子（核心）

```
净买入量 = Σ(买入股数) - Σ(卖出股数)  [5分钟时间窗口]
交易驱动因子 = (净买入量 × 价格敏感系数) ÷ 流通股本
```

**数学含义**：
- 净买入量占流通股本的比例，决定价格变动方向和幅度
- 价格敏感系数（0.005）控制"杠杆效应"
- 例如：净买入流通股本的1%，价格上涨0.5%

**经济学原理**：
> 当买盘大于卖盘时，需求超过供给，价格上涨；反之下跌。这正是真实股市的供需法则。

#### 2.2.2 随机因子（噪声）

```
随机因子 = (Math.random() - 0.5) × 2 × 波动率 × 0.3
```

**作用**：
- 模拟市场中的"噪声交易"
- 反映信息不对称、情绪波动等不可预测因素
- 防止价格走势过于线性

#### 2.2.3 趋势因子（动量）

```
趋势因子 = 股票趋势值 × 0.05
```

**作用**：
- 模拟真实市场的"动量效应"
- 正趋势：股票有上涨惯性
- 负趋势：股票有下跌惯性

## 三、风险控制机制

### 3.1 价格熔断

```
单次最大涨跌幅 = 当前价格 × 10%
新价格 ∈ [当前价格 - 10%, 当前价格 + 10%]
```

**类比真实市场**：
- A股：涨跌停板制度（±10%）
- 美股：熔断机制（7%/13%/20%三级）
- 本系统：单次更新熔断，防止极端波动

### 3.2 绝对价格区间

```
价格下限 = 基础价 × 10%  （防止归零）
价格上限 = 基础价 × 500% （防止泡沫）
```

### 3.3 单用户交易上限

```
单次最大交易量 = 流通股本 × 5%
```

**防操纵原理**：
- 限制单个用户对价格的影响权重
- 防止"大户"单边拉升或砸盘
- 保护中小投资者利益

---

## 四、流动性保障

### 4.1 做市商机制

当市场无人交易时，系统自动注入虚拟交易量：

```
做市交易量 = 50 ~ 150 股（随机）
```

**作用**：
- 避免价格因无交易而"僵死"
- 提供基础的市场深度
- 模拟真实市场的"流动性提供者"

### 4.2 时间窗口设计

```
交易量统计窗口 = 5分钟
价格更新频率 = 1分钟
```

**设计考量**：
- 5分钟窗口：平衡灵敏度与稳定性
- 1分钟更新：给予用户反应时间
- 滑动窗口：持续反映最新交易情况

## 五、交易流程详解

### 5.1 买入流程

```
用户发起买入请求
    ↓
检查1：股票是否存在且活跃
    ↓
检查2：交易量是否超过上限（5%流通股本）
    ↓
检查3：可用股份是否充足
    ↓
检查4：用户贡献点是否充足
    ↓
执行交易：
  ├── 扣除贡献点
  ├── 减少可用股份
  ├── 增加用户持仓
  └── 记录交易
    ↓
更新市场：
  └── 累计交易量 → 影响下次价格
```

### 5.2 卖出流程

```
用户发起卖出请求
    ↓
检查1：股票是否存在且活跃
    ↓
检查2：交易量是否超过上限
    ↓
检查3：用户持仓是否充足
    ↓
执行交易：
  ├── 增加贡献点
  ├── 增加可用股份
  ├── 减少用户持仓
  └── 记录交易
    ↓
更新市场：
  └── 累计交易量 → 影响下次价格
```

## 六、盈亏计算

### 6.1 持仓成本

```
平均成本 = (原持有股数 × 原成本 + 新买入股数 × 买入价) ÷ 总持有股数
```

**示例**：
- 第一次买入100股，价格10元，成本 = 1000元
- 第二次买入100股，价格12元，成本 = 1200元
- 平均成本 = (1000 + 1200) ÷ 200 = 11元/股

### 6.2 浮动盈亏

```
当前市值 = 持有股数 × 当前价格
浮动盈亏 = 当前市值 - (持有股数 × 平均成本)
盈亏比例 = 浮动盈亏 ÷ 总成本 × 100%
```

## 七、市场博弈分析

### 7.1 投资策略

| 策略 | 原理 | 风险 |
|:---|:---|:---|
| **追涨** | 买入上涨股票，期待继续上涨 | 可能接盘高位 |
| **抄底** | 买入下跌股票，期待反弹 | 可能抄在半山腰 |
| **波段** | 低买高卖，赚取差价 | 需要精准判断 |
| **持有** | 长期持有，等待趋势 | 时间成本高 |

### 7.2 市场情绪传导

```
用户A买入 → 净买入量增加 → 价格上涨 → 吸引更多买入 → 价格继续上涨
    ↑                                              ↓
    ←────────── 正反馈循环（牛市）←─────────────────

用户B卖出 → 净买入量减少 → 价格下跌 → 引发恐慌抛售 → 价格继续下跌
    ↑                                              ↓
    ←────────── 负反馈循环（熊市）←─────────────────
```

### 7.3 博弈均衡

在交易驱动模型下，市场会趋向**动态均衡**：

```
价格上涨 → 卖盘增加（获利了结）→ 净买入量下降 → 价格回落
价格下跌 → 买盘增加（抄底入场）→ 净买入量上升 → 价格回升
```

## 八、与真实股市的对比

| 特性 | 真实股市 | 虚拟股市 |
|:---|:---|:---|
| **定价机制** | 竞价撮合 | 交易驱动+随机 |
| **流动性** | 做市商/订单簿 | 系统做市商 |
| **涨跌限制** | 涨跌停/熔断 | 10%单次熔断 |
| **信息透明** | 公告/财报 | 完全透明 |
| **风险** | 真实资金损失 | 无真实风险 |
| **监管** | 证监会 | 代码硬约束 |

## 十、总结

玄剑虚拟股市通过**交易驱动模型**实现了供需决定价格的核心机制，同时引入：

1. **熔断机制**：防止极端波动
2. **交易上限**：防止市场操纵
3. **做市商**：保障流动性
4. **随机噪声**：模拟市场不确定性

这套机制让用户的每一次交易都能影响市场，创造了真实的市场博弈体验，同时通过多重安全机制保护了市场的公平性和稳定性。
